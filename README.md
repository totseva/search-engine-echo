# search-engine-echo   

Step 1 Research & Wireframing

This first step focuses on defining the UX foundations for a new search engine. The goal is to understand user needs, help users search  without distractions, minimalistic interface, yet helping and suggesting possible recommendations.

I’m analyzing:
Existing search engines and their UX patterns (Google, Yandex, Yahoo and expiring ideas of the Wayback Machine, etc.)
How users behave when they don’t have a clear query
Common UX pain points: clutter, lack of guidance
Patterns for minimalism and trust (privacy, clarity)

I’m trying to achieve:
A minimal, calm search experience 
Search engine with links to images, videos, maps, news
Possible gamification and discovery mechanics

Step 2. Implement the ideas into HTML/CSS

Overview:
Echo is a minimalistic, calming search engine designed to reduce visual clutter while providing all the essential links and tools in one accessible location. The interface is intentionally spacious, with a thoughtful hierarchy and subtle cues to guide users’ attention.

Key UX Principles:

- Calming, non-distracting color palette: Soft, neutral tones that make the page visually restful and focus attention on the search task.

- Minimalistic layout: The main page features a prominent search input, a logo, and a small set of essential action buttons, avoiding visual noise.

- Useful links in one place: Important links such as Email, Images, Maps, News, Weather, and Profile are easily accessible without overwhelming the user.

- Idea suggestion via “Trending Today”: A dedicated button encourages exploration and gives users inspiration for searches they might not think of, subtly gamifying the experience.

- Accessibility features: Proper HTML semantics, ARIA labels, and mobile-friendly layout ensure that the site is usable for all users, including screen reader users.

- Mobile optimization: On mobile, the interface is simplified — the header and footer are hidden to maximize focus on the search input, while spacing and sizing are adapted for smaller screens.

- Visual hierarchy: Desktop typography is intentionally sized — header links use larger fonts for high-frequency actions, while footer links use smaller fonts for secondary actions.

- Breathing space: Adequate spacing between links, buttons, and sections creates a clean visual rhythm and reduces cognitive load.

- Interactive elements: Buttons have hover, shadow when hover, reinforcing interactivity without distraction.

- Name philosophy: the “echo” ensures every search opens the door to many possibilities. User types one thing, it returns many.
